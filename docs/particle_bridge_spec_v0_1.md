# Particle Bridge Spec v0.1 (Tier B)

## 1) Purpose and Scope
This is a **Tier B specification**, not an operational Tier A engine. It defines bounded, explicit future particle-sector bridges without emitting Tier A observables.

**Tier A implemented today**: Phase‑2 cosmology runs, cluster evidence packets, quantum boundary packet (controlled/self‑consistent), and deterministic residual metrics. **Not implemented**: particle‑sector operators or Standard Model derivations.

## 2) Ontology
- Particles are localized excitations interacting with a finite‑bandwidth viscoelastic metric background.
- Localization, propagation, and decoherence are constrained by metric response rather than empty vacuum.
- Distinguish:
  - **Tier A operational observables**: deterministic, hashable evidence packets.
  - **Tier B UV/microstate derivations**: structural claims, not engine outputs.

## 3) Existing locked bridges
Current Tier A quantum bridge (already implemented):
- `OP_QUANTUM_DECOHERE_001`
- Self‑consistent law: $m^{-2/3}$
- Controlled $\omega$ policy as experimental interface

These motivate the particle bridge but **do not complete it**.

## 4) Tier B structural claims
- 3‑loop anomaly residue $R \sim 1.15428$ as a UV regulation anchor.
- Vacuum saturation radius $r_{sat}$ as a saturation‑boundary concept.
- Localization / finite‑momentum / finite‑bandwidth interpretation of excitations.

These are Tier B only and **not emitted** by current engine operators.

## 7) Falsification standards
If promoted to Tier A, operators must:
- Emit deterministic, hashable observables.
- Avoid hidden fit parameters.
- Declare baseline‑vs‑derived constants.
- Survive Tier A audit (offline‑safe, reproducible).

## 8) Boundaries
- This spec does **not** claim a derived Standard Model Lagrangian.
- This spec does **not** add Tier A observables.
- This spec defines a bounded path to future operators.
